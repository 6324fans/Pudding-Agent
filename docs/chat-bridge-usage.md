# 聊天桥接使用说明

> 更新时间：2026-05-30

Pudding-Agent 已提供微信和飞书聊天桥接 MVP。入口在设置页的“聊天桥接”Tab。

## 能力范围

- 微信：扫码登录、轮询接收文本消息、回发文本回复。
- 飞书：接收 webhook 文本事件、回复原消息，失败时按 chat_id 发送新消息。
- 路由：按聊天窗口生成 `routeKey`，自动绑定到 Pudding session。
- 可视化：设置页可查看渠道状态、最近事件、会话路由、入站/出站计数，并可对路由执行“新会话”和“解绑”。
- 安全：支持远程配对保护，开启后新聊天窗口必须先发送 `/pair <配对码>`。
- 审批：工具权限请求会推送到绑定 route，可在微信/飞书中用 `/OK` 或 `/NO` 处理。
- 指令：远程聊天内支持 `/pair`、`/help`、`/status`、`/new`、`/reset`、`/stop`、`/P`、`/OK`、`/NO`。

## 微信接入

1. 打开设置页，进入“聊天桥接”。
2. 展开“微信”渠道配置，确认 Base URL、Bot Type、Channel Version。
3. 点击“登录”，用手机微信扫码。
4. 点击“我已扫码”，直到状态变为“已连接”。
5. 启动桥接后，微信发来的文本会自动进入绑定的 Pudding 会话，Pudding 回复会回发微信。

## 飞书接入

1. 在飞书开放平台创建应用，准备 App ID 和 App Secret。
2. 在 Pudding-Agent 设置页的“飞书”渠道中填入 App ID、App Secret。
3. 如配置了 Verification Token，也填入同一值。
4. 将设置页显示的 webhook URL 配置到飞书事件订阅中。
5. 当前 webhook 监听在 `127.0.0.1`，真实飞书回调需要公网可访问地址。可以通过内网穿透或反向代理转发到本机端口。
6. 飞书私聊机器人后，文本消息会进入 Pudding 会话，回复会优先 reply 原消息。

## 远程指令

| 指令 | 作用 |
| --- | --- |
| `/pair <配对码>` | 配对当前聊天窗口 |
| `/help` | 查看可用指令 |
| `/status` | 查看当前渠道、路由、绑定会话和最近错误 |
| `/new` | 为当前聊天窗口新建并绑定一个 Pudding 会话 |
| `/reset` | 解除当前聊天窗口的会话绑定 |
| `/stop` | 中止当前绑定会话正在执行的任务 |
| `/P` | 查看当前绑定会话最新待审批项 |
| `/OK` | 批准当前绑定会话最新待审批项 |
| `/NO` | 拒绝当前绑定会话最新待审批项 |

## 配对保护

设置页的“远程配对保护”默认可手动开启。开启后：

- 每个新的聊天 route 第一次发普通任务时，会收到配对提示。
- 用户需要发送 `/pair <配对码>`，配对成功后才能让消息进入 Pudding 会话。
- 配对码可以在设置页刷新；刷新后不影响已配对 route。
- 路由列表会显示“可信”或“未配对”，并可对已可信 route 执行“取消信任”。
- 同一个 Pudding 会话默认只绑定一个聊天 route；重新绑定时旧 route 会被解除会话绑定。

## 当前限制

- 只支持文本消息；图片、文件、语音、视频暂未接入。
- 飞书 Encrypt Key 加密事件暂未解密；如果开启加密，当前版本会拒绝事件并提示。
- 远程审批已支持最新待审批项处理；更复杂的多审批队列、审批超时、仅 owner 可审批仍需增强。
- 已有 route 配对码；allowlist、管理员分级和危险操作二次确认仍需继续补。不要把 webhook 直接暴露给不可信网络。
- 微信登录态过期、验证码登录、限流退避仍需继续增强。
